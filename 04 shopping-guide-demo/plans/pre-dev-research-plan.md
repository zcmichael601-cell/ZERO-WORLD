### 搜索场景（5 天：5/15、5/18 - 5/21）

#### 搜索 D1（5/15 周五）：Query 理解层

**研究目标：** 用户输入的文字到变成检索信号，中间经过了哪些处理？当前对自然语言 Query 的处理能力边界在哪里？

| 调研维度 | 关键问题 |
|---------|---------|
| 分词与标准化 | 使用什么分词策略？是否有品牌词/型号词专用词典？如何处理英中混写（「iPhone15黑色」）？ |
| Query 改写 | 有无同义词扩展（「手机」→「智能手机/手机壳」）？拼写纠错模型还是规则？改写后是否保留原始 Query 并行检索？ |
| 意图识别 | 识别哪些意图类别（品类/场景/比价/闲聊）？模型结构（分类模型还是生成模型）？线上准确率？ |
| Query → 类目映射 | 用户 Query 如何落到商品类目树？是多分类模型还是规则匹配？类目覆盖率？多类目 Query 如何处理？ |
| 复合 Query 能力 | 「给 8 岁男孩买 200 元以内的运动礼物」这类多条件 Query，当前能拆解哪些槽位？哪些丢失？ |

**输出：** Query 理解层流程图 + 能力边界说明（能处理什么 / 丢什么）

---

#### 搜索 D2（5/18 周一）：索引与召回层

**研究目标：** 候选商品从哪里来？有多少路召回通道？各通道的定位和实际贡献？

| 调研维度 | 关键问题 |
|---------|---------|
| 索引类型 | 倒排索引（关键词）和向量索引（语义）各覆盖多少 SKU？向量更新频率（实时/T+1）？ |
| 向量化方案 | 商品 Embedding 和 Query Embedding 用什么模型生成？维度？是否做了领域适配微调？ |
| 召回通道 | 有哪些通道（关键词/语义向量/热销/个性化协同过滤/价格区间）？每路 beam size？总召回量？ |
| 通道权重 | 各通道最终贡献多少比例的曝光商品？权重是固定的还是动态调整？ |
| 召回过滤 | 库存不足/违规/下架商品在召回阶段过滤还是排序阶段过滤？ |
| 延迟数据 | 召回阶段 P95 耗时？是否是搜索链路的主要瓶颈？ |

**输出：** 召回层架构图 + 各通道特性与贡献比例表

---

#### 搜索 D3（5/19 周二）：排序层（粗排 + 精排）

**研究目标：** 候选商品经过哪些模型决定最终排序？特征体系的完整性如何？

| 调研维度 | 关键问题 |
|---------|---------|
| 粗排（L1）| 模型结构（线性/浅层 DNN）？输入特征数量？候选从几千缩到多少？P95 延迟？ |
| 精排特征-用户侧 | 短期行为（近 N 次点击/加购/下单）、长期偏好（品类/价格区间偏好）、实时 Session 行为各有哪些特征？ |
| 精排特征-商品侧 | 类目/品牌/价格/评分/销量/库存/上架时间等结构化特征覆盖率？图文 Embedding 特征是否已入精排？ |
| 精排特征-交叉 | Query 与商品 Title 的语义相似度是否作为特征？Query × 用户历史的交叉特征有哪些？ |
| 精排模型结构 | DNN / Wide&Deep / DCN / Transformer？参数量级？在线推理 P95 延迟？ |
| 优化目标 | 主优化 CTR 还是 GMV 还是多目标？各目标的权重如何确定？ |
| 训练数据 | 正负样本定义（曝光未点击 = 负样本？）？天级更新还是更高频？ |

**输出：** 排序层模型概览 + 特征工程分层说明（用户/商品/交叉/Query 四层）

#### 搜索 D4（5/20 周三）：重排 + 运营干预

**研究目标：** 模型排序结果是否被人工或规则二次调整？干预规则是否会影响 AI 导购结果？

| 调研维度 | 关键问题 |
|---------|---------|
| 重排-多样性 | 同品牌/同店铺商品是否限制连续出现？打散策略是规则还是模型？ |
| 重排-类目配额 | 不同类目商品在结果页有没有配额限制？谁来配置？ |
| 白名单/置顶 | 哪类商品可被运营置顶（自营/品牌旗舰/大促主推）？触发方式（人工配置 / 规则自动）？覆盖多少比例的 Query？ |
| 黑名单/降权 | 哪类商品会被降权或屏蔽（差评/违规/竞品）？规则触发还是人工审核？ |
| 大促特殊干预 | 618/双11 期间是否有专项干预逻辑覆盖正常排序规则？AI 导购结果是否也要服从？ |
| 干预透明度 | 运营干预是否对用户可见？AI 推荐结果是否需要遵守与搜索相同的干预规则？ |

**输出：** 运营干预规则清单 + 对 AI 导购的影响评估（哪些规则必须遵守/哪些可豁免）

### 广告 SPU 场景（3 天：6/01 - 6/03）

> 重点：SPU（Standard Product Unit）项目，即广告场景下的商品聚合与标准化

#### 广告 D1（6/01 周一）：SPU 项目背景与整体方案

**研究目标：** SPU 是什么？为什么广告场景要做 SPU 而不是 SKU 级别的投放？

| 调研维度 | 关键问题 |
|---------|---------|
| SPU vs SKU | SPU 的定义边界（同款不同规格的商品聚合逻辑）？一个 SPU 下挂多少 SKU？ |
| 项目目标 | SPU 项目要解决广告场景的哪些问题（重复曝光 / 质量低 SKU 抢量 / 商品信息标准化）？ |
| SPU 生成 | SPU 如何从商品库中构建（规则匹配 / 图文相似度 / 类目属性对齐）？覆盖率目前多少？ |
| SPU 维护 | SPU 的增量更新机制？当某个 SKU 下架，SPU 如何处理？ |
| 项目现状 | 目前上线情况（全量 / 部分品类 / 灰度）？主要负责团队？ |

**输出：** SPU 项目背景与整体方案说明

---

#### 广告 D2（6/02 周二）：SPU 在广告链路中的应用

**研究目标：** SPU 如何改变广告的召回、排序和展示？

| 调研维度 | 关键问题 |
|---------|---------|
| SPU 级广告召回 | 广告主是针对 SPU 投放还是 SKU？Query-SPU 匹配策略和 Query-SKU 有何不同？ |
| SPU 出价与排序 | eCPM 在 SPU 粒度如何计算？多个 SKU 对应同一 SPU 时如何处理出价竞争？ |
| SPU 质量分 | SPU 级别的质量分包含哪些因子（图文质量/评价聚合/描述完整性）？如何影响排名？ |
| 展示形式 | 广告结果展示的是 SPU 主图还是最优 SKU？主图/主 SKU 选取逻辑？ |
| 效果数据 | SPU 广告 vs SKU 广告的 CTR / CVR / ROI 对比数据（如有）？ |

**输出：** SPU 广告链路流程图 + 与 SKU 级广告的核心差异对比

#### 广告 D3（6/03 周三）：SPU 与 AI 导购结合点 + 综合产出 → M3

**上午：SPU 对 AI 导购的价值**

| 调研维度 | 关键问题 |
|---------|---------|
| 减少重复推荐 | AI 导购返回多个商品时，是否应该在 SPU 粒度去重（避免同款不同颜色占满推荐位）？ |
| SPU 描述作为 RAG 素材 | SPU 的标准化描述/属性是否比单个 SKU 的 Title 更适合作为 LLM 推荐的输入素材？ |
| 广告与 AI 推荐融合 | 在 AI 导购结果中，广告 SPU 和自然推荐 SPU 如何隔离或融合？现有技术支撑？ |

**下午：现状调研综合产出**

1. **全链路架构图**：搜索 / 推荐 / 广告 SPU 三条主线的完整流程图

4. **改造难点标注**：哪里是大改（>1 个月）/ 小改（<1 周）/ 直接复用
