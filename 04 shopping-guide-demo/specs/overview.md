## 一、产品定位

### 1.1 产品背景

AI 大模型正在重构购物体验入口。Amazon Rufus 已证明：嵌入购物流程的对话式 AI 能将用户购买转化率提升 60%。国内阿里、字节也在快速布局，但普遍存在三个问题：推荐混入广告（信任低）、通用品类深度不足、追问流程生硬。

我们的机会：在 AI 导购普及前，占据**垂直品类 × 无广告干扰 × 高信任度**的差异化定位。

### 1.2 一句话定位

**「帮你把要买的东西说清楚，让算法帮你找到最合适的那一个」**

像对朋友一样描述需求，系统理解意图后推荐 + 解释，用户只需要做最终决策。

### 1.3 目标用户

| 用户类型 | 描述 | 核心诉求 |
|---------|------|---------|
| **决策困难型** | 面对大量商品不知道怎么选，纠结很久仍难决定 | 帮我缩小选项，给我一个推荐理由 |
| **需求模糊型** | 有购物意图但表达不清（"给妈妈买个礼物"） | 帮我把需求问清楚，再给结果 |
| **效率优先型** | 知道要什么，但懒得一条条筛选 | 快速、准确、理由充分 |
| **研究型买家** | 大额商品（家电/3C）购买前习惯做大量调研 | 帮我对比参数，用人话解释差异 |

### 1.4 核心价值主张

1. **无广告推荐**：结果由算法决定，明确标注，建立信任
2. **对话式表达**：用户不需要会「搜索技巧」，自然语言即可
3. **三链路保障**：传统搜索兜底 + LLM 提升质量 + Fallback 防降级，系统稳定可靠
4. **追问不尴尬**：系统追问像朋友聊天，不是填表单

### 1.5 成功指标

| 指标 | 定义 | Demo 目标值 | v1.0 目标值 |
|------|------|-----------|-----------|
| 对话完成率 | 用户开始对话后看到推荐结果的比例 | >60% | >80% |
| 推荐点击率 | 看到推荐商品后点击查看的比例 | >30% | >50% |
| 意图识别准确率 | 系统理解用户意图正确的比例 | >70% | >85% |
| 追问接受率 | 系统发出追问后用户继续回答的比例 | >50% | >70% |
| LLM 链路成功率 | LLM 请求成功（非 fallback）的比例 | - | >95% |
| P95 响应时延 | 传统链路 / LLM 链路（流式首字） | <200ms / <1s | <200ms / <800ms |

## 二、用户使用场景

### 2.1 主场景路径

**场景 A：需求模糊，需要引导**
1. 用户打开 App，进入对话页
2. 用户输入模糊需求："帮我找个母亲节礼物"
3. 系统识别意图为「guided_shopping」，触发 LLM 追问流程
4. 系统问："妈妈平时喜欢什么？预算大概多少？"
5. 用户补充："她喜欢护肤，预算 300 以内"
6. 槽位填满 → 触发搜索 + LLM 推荐生成
7. 展示 2-3 个推荐商品 + 推荐理由
8. 用户点击感兴趣的商品 → 跳转详情页

**场景 B：需求明确，快速搜索**
1. 用户输入："Nike Air Max 42 码黑色"
2. 系统识别意图为「simple_search」，直接触发传统搜索链路
3. <200ms 返回商品列表
4. 用户浏览 → 点击购买

**场景 C：选择困难，需要对比**
1. 用户在搜索结果页面，对多个商品举棋不定
2. 用户输入："这两个有什么区别？哪个适合我？"
3. 系统识别意图为「comparison」，调用 LLM 对比生成
4. 展示关键差异点（人话版）+ 建议

### 2.2 用户旅程地图

| 阶段 | 用户行为 | 用户想法 | 痛点 | 机会点 |
|------|---------|---------|------|-------|
| 进入 | 打开 App | "我要买东西" | 不知道从哪开始说 | 引导性首屏提示 |
| 表达需求 | 输入/说出需求 | "说不清楚怎么办" | 关键词搜索不精准 | 自然语言接收，追问补全 |
| 获取推荐 | 等待结果 | "这个靠谱吗" | 结果太多/太少/混广告 | 少而精 + 给理由 + 无广告标注 |
| 决策 | 对比、思考 | "我该选哪个" | 参数看不懂，比较累 | Help Me Decide，人话解释 |
| 购买 | 点击下单 | "买完就算了" | 跳来跳去 | 链路内闭环（不跳出） |
